# 如何管理FAQ和群聊

> **适用场景**：hackathon活动期间需要管理参与者群聊、收集整理FAQ、发布各类通知。
> **前置条件**：已完成指南文档编写（参考 10_1_如何写指南文档）。

---

## 第一步：收集FAQ

### FAQ收集渠道

| 渠道 | 说明 | 收集方式 |
|------|------|----------|
| **选手群** | 最主要的FAQ来源 | 定期翻看群聊记录，提取高频问题 |
| **报名表反馈** | 报名阶段的问题 | 查看报名表的"疑问"字段 |
| **往届活动** | 历史经验 | 从往届活动的FAQ中筛选适用项 |
| **志愿者反馈** | 现场遇到的问题 | 志愿者培训时收集 |
| **主动预判** | 提前想可能会问的问题 | 团队头脑风暴 |

### FAQ收集时间线

- **活动前4周**：收集报名相关FAQ
- **活动前2周**：收集参赛规则、组队相关FAQ
- **活动前1周**：收集交通、住宿、餐饮相关FAQ
- **活动当天**：收集现场执行相关FAQ（实时更新）

## 第二步：分类整理FAQ

### FAQ分类框架

| 分类 | 典型问题 |
|------|----------|
| **报名相关** | 如何报名？报名截止时间？可以跨校组队吗？ |
| **参赛规则** | 可以用开源代码吗？项目主题有限制吗？团队人数限制？ |
| **技术相关** | 提供什么API？可以用自己的设备吗？WiFi密码是什么？ |
| **组队相关** | 如何找到队友？可以一个人参加吗？可以中途换队吗？ |
| **提交相关** | 提交什么内容？截止时间？可以修改提交吗？ |
| **评审相关** | 评分标准是什么？评审流程？ |
| **后勤相关** | 住宿怎么安排？有餐饮吗？停车方便吗？ |
| **其他** | 活动周边？证书？社交活动？ |

```
### Q1：[问题]
**A**：[回答]

### Q2：[问题]
**A**：[回答]

```

### FAQ文档模板示例

```markdown
# [活动名称] 常见问题（FAQ）

最后更新：20XX年X月X日

---

## 一、报名相关

### Q1：如何报名参加？
**A**：请访问 [报名链接]，填写报名表并提交。报名截止时间为X月X日。

### Q2：可以跨校组队吗？
**A**：可以！我们鼓励跨校、跨专业组队。你可以在组队需求墙上寻找队友。

### Q3：报名后多久能收到确认？
**A**：我们会在3个工作日内审核你的报名信息，并通过邮件/短信通知审核结果。

## 二、参赛规则

### Q1：项目主题有限制吗？
**A**：本届活动设有[X个主题方向]，但不限制具体项目方向，鼓励创新。

### Q2：可以使用开源代码吗？
**A**：可以使用，但必须在项目说明中标注引用来源。核心功能需团队自主开发。
```

## 第三步：制定FAQ维护规范

### FAQ维护流程

```
发现新问题
    ↓
判断是否为高频问题（≥3人问同一问题）
    ↓
是 → 撰写FAQ回答 → 审核 → 发布 → 通知群内
否 → 直接回复提问者
    ↓
定期review已有FAQ → 更新过时信息
```

### FAQ维护checklist

- [ ] 每天检查群聊中的新问题
- [ ] 每周更新一次FAQ文档
- [ ] 重大变更（如时间调整）立即更新FAQ并通知
- [ ] FAQ文档中标注最后更新时间
- [ ] 活动结束后整理最终版FAQ存档

## 第四步：制定群规则

### 群公告模板

```
📢 [活动名称] 选手群规则

欢迎加入[活动名称]选手群！为了让大家有良好的交流体验，请遵守以下规则：

1️⃣ 【实名制】请将群昵称改为：姓名-学校/公司
2️⃣ 【禁止广告】禁止发布与活动无关的广告、推广信息
3️⃣ 【文明交流】尊重他人，禁止人身攻击、歧视性言论
4️⃣ 【提问规范】提问前请先查看FAQ文档，避免重复提问
5️⃣ 【信息保密】不要在群内分享他人隐私信息
6️⃣ 【重要通知】请关注群公告，重要通知以群公告为准

📋 FAQ文档：[链接]
📅 活动日程：[链接]
❓ 联系方式：[联系人] [联系方式]

祝大家比赛愉快！🎉
```

### 群规则执行要点

- **入群即发**：新人入群时自动发送群规则
- **定期提醒**：每周在群内提醒一次群规则
- **违规处理**：①第一次警告私聊；②第二次群内警告；③第三次移出群聊
- **管理员配置**：每群至少2名管理员，覆盖不同时段

## 第六步：群聊日常管理

### 群聊管理日历

| 时间 | 管理动作 | 负责人 |
|------|----------|--------|
| 每天 | 检查群内问题，及时回复 | 群管理员 |
| 每周一 | 发送本周重要事项提醒 | 群管理员 |
| 报名截止前3天 | 发送报名提醒 | 群管理员 |
| 活动前1天 | 发送日程和注意事项 | 群管理员 |
| 活动当天 | 实时发布现场通知 | 群管理员 |
| 活动结束后 | 发送感谢通知和后续安排 | 群管理员 |

### 群聊管理技巧

- **置顶重要消息**：将FAQ链接、日程表等置顶
- **及时回复**：问题在2小时内回复（工作时间）
- **引导讨论**：适时发起话题讨论，保持群活跃度
- **整理精华**：将有价值的讨论内容整理成精华消息
- **控制节奏**：避免群内信息过于密集或过于冷清

## 常见问题

**Q：群太多管不过来怎么办？**
A：①控制群数量，不要建太多群；②每个群指定2-3名管理员；③使用群机器人自动回复常见问题。

**Q：有人在群里发广告怎么办？**
A：①立即删除消息；②私聊警告；③严重者移出群聊；④在群规则中明确禁止广告。

**Q：FAQ回复不及时怎么办？**
A：①提前准备FAQ文档覆盖大部分问题；②设置群关键词自动回复；③安排值班管理员覆盖不同时段。

**Q：如何处理群内的负面情绪？**
A：①及时私聊了解情况；②公开回复时保持专业和友善；③如果是合理的抱怨，及时改进并反馈；④避免在群内争论。

### 真实案例：历史活动 FAQ问答

**Q：谁可以来参加？对参与者的要求是什么？**
A：欢迎任何背景参与者。不只是一次集中开发，更是让产品接受真实市场检验的舞台。

**Q：申请流程是什么？**
A：确定身份（参赛选手/合作方/志愿者/游客）→填写申请问卷。无面试，申请材料是唯一录取依据。分批次录取并发确认邮件，收到邮件需在给定时间内确认，否则失去参赛资格。

**Q：我们能以团队形式申请吗？**
A：所有申请以个人为单位，已有队员不作为录取考量因素。有合作伙伴需分别申请。

**Q：如果我就是过来逛逛，可以参与哪些活动？**
A：第一天→合作方展会和开幕式；中间一天→选手互动或Workshop；最后一天→游园会打榜投票+闭幕式。

**Q：比赛期间是否必须全程在场？**
A：不强制，但强烈建议全程参与。

**Q：比赛是必须新开项目吗？**
A：可自带项目，但评选标准会严格区分自带项目和现场组队项目（不同评价维度）。自带项目需在报名初期声明。

**Q：允许个人参赛吗？什么时候组队？**
A：要求2-6人组队参赛。可在选手群发布招募信息。

**Q：赛事举办的时间与地点？**
A：4天活动，第1天签到+开幕式+展会，第2-3天开发+Workshop，第4天游园会+路演+颁奖+After Party。

**Q：参赛期间有餐饮供应吗？**
A：全程为所有选手提供餐饮（早餐到宵夜）。

**Q：是否有路费住宿补助？**
A：路费补助往返最高200元。住宿权益覆盖2晚，酒店随机分配。

## 历史教训

在hackathon活动中，FAQ和群聊管理建议：
- **群机器人**：使用微信群机器人或飞书机器人自动回复FAQ，减少管理员工作量
- **FAQ文档**：使用在线文档（飞书/腾讯文档），方便实时更新和多人协作
- **分群管理**：建议建"选手大群"+"各小组群"+"志愿者群"+"嘉宾群"，信息分流
- **现场FAQ**：在场地设置FAQ展板/易拉宝，减少重复提问
- **即时通讯**：hackathon节奏快，群通知要简洁明了，重要信息重复发2-3次确保覆盖
