# 14_2 如何做活动直播

> 适用场景：活动开幕式、路演、Demo Day等重要环节的线上直播。
> 前置条件：[14_1_如何搭建活动官网](14_1_如何搭建活动官网.md)

## 第一步：选择直播平台

### 主流直播平台对比

| 平台 | 延迟 | 画质 | 互动功能 | 适用场景 | 费用 |
|---|---|---|---|---|---|
| B站直播 | 低 | 高 | 弹幕、礼物 | 技术/学生社区 | 免费 |
| 微信视频号 | 低 | 高 | 评论、点赞 | 微信生态传播 | 免费 |
| 小红书直播 | 低 | 高 | 评论、点赞 | 年轻用户 | 免费 |
| 抖音直播 | 低 | 高 | 弹幕、连麦 | 泛用户传播 | 免费 |
| 腾讯会议 | 中 | 中 | 举手、聊天 | 内部活动/小型 | 免费（限时） |
| YouTube Live | 高 | 高 | 聊天 | 海外观众 | 免费 |

### 推荐组合方案

| 方案 | 平台组合 | 适用场景 |
|---|---|---|
| 国内为主 | B站 + 视频号 | 面向国内年轻用户 |
| 小红书为主 | 小红书直播 | 面向小红书社区用户 |
| 纯内部 | 腾讯会议/飞书会议 | 内部活动、小型活动 |

### 开播前准备

- [ ] 注册并完成平台实名认证
- [ ] 创建直播间，设置标题、封面、分类
- [ ] 测试推流地址和推流密钥
- [ ] 提前3天在社群/官网预告直播时间
- [ ] 准备直播间公告（活动信息、互动规则）

## 第三步：安装和配置OBS

### 3.1 下载安装

1. 访问 https://obsproject.com/ 下载OBS Studio
2. 安装完成后打开，运行"自动配置向导"
3. 选择"优先使用流媒体优化"

### 3.2 获取推流地址

以B站直播为例：
1. 登录B站 → 直播中心 → 开播设置
2. 选择"第三方推流"
3. 复制"服务器地址"和"串流密钥"

### 3.3 OBS推流设置

1. 打开OBS → 设置 → 推流
2. 服务选择"自定义"
3. 服务器：粘贴B站的服务器地址
4. 串流密钥：粘贴B站的串流密钥
5. 点击"确定"

### 3.4 OBS输出设置

1. 设置 → 输出 → 输出模式选择"高级"
2. 编码器：优先选择"硬件编码"（如NVENC、QuickSync）
3. 码率：
   - 720P：2500-4000 Kbps
   - 1080P：4000-6000 Kbps
4. 关键帧间隔：2秒

### 3.5 OBS视频设置

1. 设置 → 视频
2. 基础（画布）分辨率：1920×1080
3. 输出（缩放）分辨率：1920×1080（如果网络好）或1280×720（网络一般）
4. 常用FPS值：30

## 第四步：配置直播场景

### 4.1 场景设计

在OBS中创建以下场景：

| 场景名称 | 用途 | 画面内容 |
|---|---|---|
| **暖场** | 直播开始前等待 | 活动主KV + 倒计时 + BGM |
| **主会场** | 开幕式、演讲 | 摄像头画面 + 角标 |
| **路演** | 项目路演 | 摄像头 + PPT画中画 |
| **采访** | 选手/嘉宾采访 | 摄像头特写 |
| **休息** | 环节间隙 | "稍后回来"画面 + BGM |
| **结束** | 直播结束 | 感谢画面 |

### 4.2 暖场场景搭建

1. 在OBS中点击"场景"下的"+"，命名为"暖场"
2. 在"来源"中点击"+" → "图像" → 选择活动主KV图片
3. 在"来源"中点击"+" → "媒体源" → 选择倒计时视频（如有）
4. 在"来源"中点击"+" → "媒体源" → 选择BGM音频文件
5. 调整各元素的位置和大小

### 4.3 主会场场景搭建

1. 创建新场景，命名为"主会场"
2. 在"来源"中点击"+" → "视频捕获设备" → 选择摄像头
3. 调整摄像头画面大小，铺满整个画面
4. 在"来源"中点击"+" → "图像" → 添加活动角标（右上角logo）
5. 如需要画中画效果：添加第二个视频源，缩小放在角落

### 4.4 路演场景搭建

1. 创建新场景，命名为"路演"
2. 添加"窗口捕获"或"显示器捕获" → 捕获PPT画面
3. 添加"视频捕获设备" → 摄像头画面（缩小放在右下角）
4. 添加"音频输入捕获" → 话筒
5. 调整布局：PPT占主画面，摄像头小窗在右下角

## 第五步：直播前测试

### 测试清单

| 测试项 | 测试方法 | 通过标准 |
|---|---|---|
| 推流连接 | 点击"开始推流" | 直播间能看到画面 |
| 画面清晰度 | 查看直播间预览 | 画面清晰，无马赛克 |
| 声音质量 | 说话测试 | 声音清晰，无杂音、无回声 |
| 网络稳定性 | 推流10分钟 | 无卡顿、无断流 |
| 场景切换 | 切换所有场景 | 切换流畅，无黑屏 |
| 画中画 | 摄像头+PPT同时显示 | 两个画面都正常 |
| BGM | 播放背景音乐 | 音量适中，不盖过人声 |
| 弹幕互动 | 发弹幕测试 | 能看到弹幕 |

### 测试时间

- **T-3天**：第一次完整测试（设备+网络+推流）
- **T-1天**：在场地进行现场测试（设备+场地网络）
- **T日开始前30分钟**：最终测试

## 第六步：直播执行

### 6.1 直播前（T-30分钟）

| 时间 | 任务 | 负责人 |
|---|---|---|
| T-30分钟 | 设备就位、开机 | 技术组 |
| T-25分钟 | OBS打开、推流测试 | 技术组 |
| T-20分钟 | 音视频测试 | 技术组 |
| T-15分钟 | 切换到暖场场景、开始推流 | 技术组 |
| T-10分钟 | 确认直播间画面正常 | 技术组 |
| T-5分钟 | 弹幕提醒"即将开始" | 运营组 |

### 6.2 直播中

| 任务 | 负责人 | 注意事项 |
|---|---|---|
| 场景切换 | 技术组 | 提前知道下一个环节，准备切换 |
| 音量控制 | 技术组 | 注意人声和BGM的音量平衡 |
| 弹幕互动 | 运营组 | 及时回复弹幕、引导互动 |
| 画面监控 | 技术组 | 注意画面是否卡顿、黑屏 |
| 备用方案 | 技术组 | 如推流中断，立即重连 |

### 6.3 直播场景切换流程

```
环节即将开始 → 提醒技术组 → 技术组准备下一个场景 → 环节开始 → 切换场景 → 环节结束 → 切换到休息场景 → 等待下一个环节
```

### 6.4 常见问题处理

| 问题 | 处理方法 |
|---|---|
| 推流中断 | 1. 检查网络 → 2. 重新推流 → 3. 切换备用网络 |
| 画面卡顿 | 1. 降低输出分辨率 → 2. 降低码率 → 3. 关闭其他程序 |
| 声音异常 | 1. 检查话筒连接 → 2. 检查OBS音频设置 → 3. 更换话筒 |
| 画面黑屏 | 1. 检查摄像头连接 → 2. 重新添加视频源 → 3. 更换摄像头 |
| 回声 | 1. 主播戴耳机 → 2. 关闭音箱 → 3. 调整话筒灵敏度 |

## 第七步：直播后处理

### 7.1 保存回放

- B站：直播结束后自动保存回放，在"创作中心"中发布
- 视频号：直播结束后在"视频号助手"中下载回放
- 如使用OBS本地录制：文件保存在OBS设置的录制路径中

### 7.2 回放剪辑

- 将回放剪辑成精彩片段（3-5分钟）
- 用于社交媒体传播（小红书、公众号）
- 提取关键演讲内容作为独立视频

### 7.3 数据统计

| 数据项 | 来源 | 用途 |
|---|---|---|
| 最高在线人数 | 直播平台后台 | 传播效果评估 |
| 总观看人数 | 直播平台后台 | 传播效果评估 |
| 互动量（弹幕/评论） | 直播平台后台 | 互动效果评估 |
| 直播时长 | 直播平台后台 | 活动记录 |

## 常见问题

**Q：网络带宽不够怎么办？**
A：降低输出分辨率（1080P→720P），降低码率（4000→2000 Kbps），或者使用有线网络替代WiFi。

**Q：多平台同时直播怎么做？**
A：使用OBS的多路推流插件（obs-multi-rtmp），或者使用Restream.io等第三方服务。

**Q：直播中有人发不当弹幕怎么办？**
A：提前设置弹幕关键词过滤，安排专人监控弹幕，及时禁言违规用户。

**Q：嘉宾/选手的PPT怎么接入直播？**
A：使用"窗口捕获"或"显示器捕获"来捕获PPT画面，或者使用采集卡连接外部电脑。

## 历史教训

> 历史活动的直播：使用OBS进行推流，主要在B站和视频号同步直播。技术组负责推流和场景切换，运营组负责弹幕互动。直播回放用于后续的宣发传播。
