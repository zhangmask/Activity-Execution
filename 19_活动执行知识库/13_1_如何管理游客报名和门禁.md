# 如何管理游客报名和门禁

> **适用场景**：黑客松活动对外开放日、Demo Day、展示环节等需要管理游客入场的场景。
> **前置条件**：请先完成 `03_场地管理.md`，确认场地容量和区域划分。

---

## 第一步：设计游客报名表

### 报名表必填字段模板

| 字段名称 | 类型 | 是否必填 | 说明 |
|---------|------|---------|------|
| 姓名 | 文本 | ✅ | 真实姓名 |
| 手机号 | 文本 | ✅ | 用于发送确认短信和应急联系 |
| 邮箱 | 文本 | ✅ | 用于发送电子票和确认通知 |
| 身份类型 | 单选 | ✅ | 选手/嘉宾/观众/志愿者/媒体 |
| 所属组织/公司 | 文本 | ❌ | 选填，用于数据分析 |
| 参加日期 | 多选 | ✅ | 如有多天活动，选择参加哪天 |
| 是否需要停车 | 单选 | ❌ | 是/否，用于场地车位规划 |
| 饮食禁忌 | 文本 | ❌ | 如提供餐饮时需要 |
| 从何渠道得知活动 | 单选 | ❌ | 社交媒体/朋友推荐/官网/其他 |

### 推荐报名工具

| 工具 | 优势 | 适用场景 |
|------|------|---------|
| 腾讯问卷 | 免费、微信生态友好 | 小型活动（<200人） |
| 金数据 | 功能丰富、支持支付 | 中型活动 |
| 活动行 | 专业活动管理平台 | 大型活动、需要票务功能 |
| 麦客表单 | 表单设计灵活 | 需要个性化表单 |
| 自建表单（Typeform等） | 完全自定义 | 有技术团队支持 |

## 第三步：建立手环颜色体系

### 手环颜色分配方案

| 手环颜色 | 适用人群 | 区域权限 | 说明 |
|---------|---------|---------|------|
| 🔴 红色 | 核心组织团队 | 全区域 + 后台工作区 | 最高权限 |
| 🟡 黄色 | 评委/嘉宾 | 主会场 + 评委室 + 休息区 | VIP通道入场 |
| 🔵 蓝色 | 参赛选手 | 主会场 + 比赛区 + 休息区 | 凭报名信息领取 |
| 🟢 绿色 | 观众/游客 | 主会场 + 展示区 | 限定区域活动 |
| 🟣 紫色 | 媒体 | 全区域 + 媒体区 | 含拍摄许可 |
| ⚪ 白色 | 志愿者/工作人员 | 全区域 + 后台工作区 | 工作职责相关区域 |

### 手环管理要点

- [ ] 提前采购足量手环（比预计人数多 20% 备用）
- [ ] 手环需防水、不易撕断（建议使用编织手环）
- [ ] 签到台按颜色分区发放
- [ ] 丢失手环需凭身份证件到服务台补办
- [ ] 活动手环不可转让，一经发现取消入场资格

## 第四步：区域权限管理

### 区域划分方案

| 区域 | 允许进入 | 入口检查方式 |
|------|---------|-------------|
| 主会场/演讲厅 | 所有持手环人员 | 手环颜色验证 |
| 比赛区/Hack区 | 红/蓝/白手环 | 手环 + 人脸识别（可选） |
| 评委室 | 红/黄/白手环 | 手环 + 门禁卡 |
| 嘉宾休息室 | 红/黄/白手环 | 手环验证 |
| 展示区 | 所有持手环人员 | 开放通行 |
| 后台工作区 | 红/白手环 | 手环 + 工作证 |
| 媒体工作区 | 红/紫/白手环 | 手环验证 |

### 区域标识制作清单

- [ ] 每个区域入口设置醒目指示牌
- [ ] 使用颜色编码与手环对应
- [ ] 设置"此区域仅限XX手环进入"的提示牌
- [ ] 关键区域安排专人值守

## 第五步：人流控制方案

### 容量计算

```
场地最大容量 = 场地面积(m²) ÷ 人均占用面积(m²)
- 坐席区：人均 1.5 m²
- 站立区：人均 0.75 m²
- 混合区：人均 1.0 m²

建议使用容量 = 场地最大容量 × 80%（预留安全余量）
```

### 分时段入场方案

| 时段 | 预计人数 | 控制措施 |
|------|---------|---------|
| 开幕前30分钟 | 嘉宾/VIP 30人 | 专用通道，提前入场 |
| 开幕前15分钟 | 选手 100人 | 选手通道快速签到 |
| 开幕时段 | 观众 200人 | 分批放行，每批50人 |
| 高峰时段 | 涌入 100人 | 启动排队系统，室内达到80%时限制入场 |

### 人流监控方法

1. **人工计数**：在每个入口安排1人使用计数器
2. **电子签到**：通过签到系统实时统计入场人数
3. **对讲机通报**：每30分钟各入口向总控报告当前人数
4. **预设预警线**：
   - 🟢 绿色：容量 0-60%（正常）
   - 🟡 黄色：容量 60-80%（关注）
   - 🔴 红色：容量 80-100%（启动分流）

## 第六步：爆满分流方案

### 分流触发条件

当室内人数达到场地容量的 **80%** 时，立即启动分流方案。

### 分流方案A：排队等候

```
1. 入口工作人员通过对讲机报告："XX入口已达到80%容量"
2. 总控确认后下达："启动分流方案A"
3. 入口开始实施"出一进一"策略
4. 安排志愿者在排队区维持秩序
5. 为等候人员提供饮用水和活动手册
6. 每10分钟通报一次预计等待时间
```

### 分流方案B：引导至备用区域

```
1. 启动备用观看区（如户外大屏直播区）
2. 引导等候人员前往备用区域
3. 在备用区域安排志愿者讲解活动进展
4. 提供线上参与链接（如有直播）
5. 室内人数降至60%后恢复正常入场
```

### 分流方案C：限时限流

```
1. 实施分时段入场，每时段30分钟
2. 发放分时段入场券（按到达顺序）
3. 优先安排以下人员：
   - 有预约的嘉宾
   - 带队老师/教练
   - 特殊需求人员
4. 在社交媒体发布实时人流信息
```

## 常见问题

**Q1：报名人数超过场地容量怎么办？**
A：设置候补名单机制。先到先得，满额后自动进入候补。活动前3天统一通知候补人员是否可以参加。也可以考虑增设线上参与通道。

**Q2：有人没有报名直接来现场怎么办？**
A：准备少量"现场报名"名额（建议为总容量的5-10%），使用手机扫码现场填写报名表后发放绿色手环入场。超出名额的礼貌引导至线上观看。

**Q3：手环被发现转让怎么处理？**
A：收回双方手环，取消双方入场资格。在报名表中提前声明此规则，签到时再次提醒。

**Q4：如何防止非授权人员进入限制区域？**
A：关键区域安排专人值守，使用与手环颜色对照的权限表。发现违规进入，礼貌引导至正确区域。

**Q5：活动当天报名系统崩溃怎么办？**
A：提前导出报名名单打印纸质版，准备Excel签到表作为备份。签到时采用手动登记+拍照记录，活动后补录数据。

## 历史教训

- 历史活动活动通常在最后一天设置对外开放的 Demo Day，游客管理是重点环节
- 建议将游客签到时间提前至开幕前1小时，避免签到拥堵影响开场
- 选手和观众的动线应尽量分开，避免在比赛期间干扰选手
- 参考历史教训：蓝色手环（选手）和绿色手环（观众）的区分最为关键
- 历史活动通常使用微信小程序进行签到，结合人脸识别提高效率
- 人流控制方面，建议设置独立的"展示区"作为缓冲区，分散主会场压力
