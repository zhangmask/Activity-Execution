# 14 网站与线上平台

> 本手册适用于活动官方网站、线上报名系统、直播平台等线上设施的建设和管理。
> 参考案例：活动主办方 历史活动

---

## 一、核心原则

1. **官网是活动的门面**：官网的品质直接影响活动的专业形象
2. **信息要及时更新**：官网信息要与最新活动安排保持同步
3. **移动端优先**：大部分用户通过手机访问，移动端体验要好
4. **SEO优化**：让目标受众能搜索到活动信息

---

## 二、官网建设

### 2.1 官网内容结构

| 页面 | 内容 | 优先级 |
|---|---|---|
| 首页 | 活动主KV、时间地点、报名入口、亮点介绍 | P0 |
| 关于 | 活动介绍、往期回顾、团队介绍 | P0 |
| 赛道 | 赛道介绍、奖项设置 | P0 |
| 嘉宾 | 嘉宾/导师/评委介绍 | P1 |
| 日程 | 活动日程安排 | P1 |
| 合作 | 赞助商/合作方展示 | P1 |
| FAQ | 常见问题 | P1 |
| 报名 | 报名入口/链接 | P0 |

### 2.2 技术选型

| 方案 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| 静态网站（Next.js/Hugo） | 快速、免费托管、SEO友好 | 需要开发能力 | 有技术团队 |
| WordPress | 易管理、插件丰富 | 需要服务器 | 长期运营 |
| Webflow | 可视化设计、无需编码 | 费用较高 | 设计驱动 |
| Canva网站 | 简单快速 | 功能有限 | 临时活动 |

### 2.3 设计规范

- 响应式设计：适配手机、平板、桌面
- 加载速度：首屏加载不超过3秒
- 品牌一致性：与活动主视觉保持一致
- 无障碍：符合WCAG基本标准

## 三、线上报名系统

### 3.1 系统选型

| 方案 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| 飞书多维表格 | 免费、易用、集成 | 自定义有限 | 中小型活动 |
| 腾讯问卷 | 免费、简单 | 功能有限 | 简单报名 |
| 金数据 | 功能丰富 | 部分功能收费 | 中型活动 |
| 自建系统 | 完全自定义 | 开发成本高 | 大型/长期活动 |

### 3.2 报名流程设计

```
访问官网 → 点击报名 → 填写报名表 → 提交 → 收到确认邮件 → 等待审核 → 审核通过 → 收到确认通知
```

### 3.3 数据管理

- 定期备份报名数据
- 敏感信息加密存储
- 数据访问权限控制
- 符合隐私保护法规

## 四、直播管理

### 4.1 直播平台选择

| 平台 | 优点 | 缺点 |
|---|---|---|
| 小红书直播 | 年轻用户多、互动强 | 功能相对简单 |
| 微信视频号 | 用户基数大、易分享 | 需要认证 |
| B站直播 | 技术社区用户多 | 需要一定粉丝量 |
| 抖音直播 | 用户基数大 | 内容审核严格 |

### 4.2 直播准备

- [ ] 直播账号认证和测试
- [ ] 推流设备准备（摄像头、采集卡、电脑）
- [ ] 网络测速（上行带宽至少10Mbps）
- [ ] 直播画面设计（OBS场景设置）
- [ ] 直播间标题和封面
- [ ] 弹幕互动人员安排
- [ ] 备用推流方案

### 4.3 直播执行

- 提前30分钟开始推流测试
- 安排专人监控画面和声音
- 及时回复弹幕互动
- 重要环节切换时提醒观众
- 直播结束后保存回放

## 五、社交媒体账号管理

### 5.1 账号矩阵

| 平台 | 用途 | 更新频率 |
|---|---|---|
| 微信公众号 | 深度内容、官方通知 | 每周1-2篇 |
| 小红书 | 日常内容、互动传播 | 每天1-3篇 |
| 微博 | 话题传播、实时互动 | 每天1-2条 |
| B站 | 视频内容、技术分享 | 每周1-2个视频 |

### 5.2 账号管理规范

- 统一账号名称和头像
- 统一话题标签（如#活动名称）
- 内容发布前审核
- 及时回复评论和私信

---

## 六、易遗漏事项

1. **域名备案**：如果使用国内服务器，域名需要提前备案（至少15天）
2. **SSL证书**：确保网站使用HTTPS
3. **CDN加速**：使用CDN提升访问速度
4. **数据分析**：接入百度统计/Google Analytics，追踪访问数据
5. **404页面**：设计友好的404页面
6. **网站地图**：提交Sitemap给搜索引擎
7. **网站备份**：定期备份网站数据
8. **版权信息**：网站底部的版权信息和ICP备案号

## 八、历史教训

> **历史活动 网站**：使用活动主办方作为官方网站，包含活动介绍、往期回顾、报名入口等。网站设计文档单独维护，使用Figma进行设计。网站反馈通过飞书文档收集。

> **历史活动 的经验**：网站仓库使用GitHub托管，设计稿使用Figma。网站图片内容（往期重要照片）需要提前整理好。
