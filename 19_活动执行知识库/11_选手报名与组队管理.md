# 11 选手报名与组队管理

> 本手册适用于活动的选手报名、组队、项目提交全流程。
> 参考案例：活动主办方 历史活动

---

## 一、核心原则

1. **报名系统要简单**：减少填写项，降低报名门槛
2. **信息收集要全面**：报名时就收集后续需要的所有信息
3. **组队要灵活**：支持个人报名和团队报名，提供组队匹配机制
4. **项目提交要标准化**：统一提交格式和截止时间

---

## 二、报名流程设计

### 2.1 报名表设计要素

| 信息类别 | 必填项 | 选填项 |
|---|---|---|
| 基本信息 | 姓名、手机号、邮箱 | 学校/公司、专业/职位 |
| 参赛信息 | 赛道选择、参赛形式（个人/团队） | 项目方向、技术栈 |
| 团队信息 | 团队名称（如团队报名） | 团队成员信息 |
| 其他 | 是否需要住宿、饮食禁忌 | 往届参赛经验、作品集 |

### 2.2 报名审核流程

1. 选手提交报名表
2. 组委会审核（资质、赛道匹配度）
3. 审核通过 → 发送确认通知
4. 审核不通过 → 发送说明和建议

### 2.3 报名审核标准

| 维度 | 审核标准 |
|---|---|
| 基本资质 | 符合参赛条件（年龄、身份等） |
| 参赛意愿 | 报名信息完整、有明确的参赛方向 |
| 技术能力 | 有相关技术背景或项目经验 |
| 团队配置 | 团队成员搭配合理（如有） |

---

## 三、组队管理

### 3.1 组队模式

| 模式 | 说明 | 适用场景 |
|---|---|---|
| 个人报名 | 一个人参赛 | 独立开发者 |
| 团队报名 | 已有团队直接报名 | 已组建团队 |
| 组队匹配 | 个人报名后通过平台匹配组队 | 需要找队友的选手 |

### 3.2 组队匹配机制

- **组队需求墙**：选手发布自己的技能和需求，寻找队友
- **组队截止时间**：明确组队截止日期，之后不可更改
- **团队人数限制**：建议2-5人一组
- **跨校/跨公司组队**：是否允许，需要提前说明

### 3.3 组队信息管理

| 信息项 | 说明 |
|---|---|
| 团队名称 | 唯一标识 |
| 团队成员 | 姓名、角色、联系方式 |
| 赛道选择 | 报名的赛道 |
| 项目方向 | 初步的项目想法 |
| 变更记录 | 成员增减、赛道变更 |

## 四、项目提交管理

### 4.1 提交内容

| 提交项 | 说明 | 格式要求 |
|---|---|---|
| 项目介绍 | 项目描述、功能、技术方案 | 文档/Markdown |
| 演示视频 | 项目Demo视频 | MP4, 3分钟以内 |
| 源代码 | 代码仓库链接 | GitHub/Gitee链接 |
| 演示PPT | 路演用PPT | 16:9, PDF备份 |
| 社交媒体 | 小红书项目发布链接 | 链接 |

### 4.2 提交时间线

| 时间节点 | 任务 |
|---|---|
| T-7天 | 发布项目提交要求和模板 |
| T-3天 | 提醒截止时间 |
| T日 xx:xx | 项目提交截止 |
| T日 xx:xx | 提交确认和反馈 |

### 4.3 提交检查清单

- [ ] 项目介绍文档完整
- [ ] 演示视频可播放
- [ ] 源代码仓库可访问
- [ ] PPT格式正确（16:9）
- [ ] 小红书笔记已发布
- [ ] 团队信息准确

## 五、选手沟通管理

### 5.1 通知邮件模板

| 场景 | 邮件主题 | 内容要点 |
|---|---|---|
| 报名确认 | [活动名称] 报名确认通知 | 确认信息、下一步指引 |
| 组队提醒 | [活动名称] 组队截止提醒 | 截止时间、组队指南 |
| 材料提交 | [活动名称] 项目提交提醒 | 提交要求、截止时间 |
| 赛事信息 | [活动名称] 赛事安排通知 | 日程、场地、注意事项 |

### 5.2 选手社群管理

- 建立选手微信群，方便沟通
- 定期发布赛事信息和提醒
- 及时回答选手疑问
- 鼓励选手之间交流和组队

---

## 六、易遗漏事项

1. **报名数据备份**：定期导出报名数据，避免系统故障导致数据丢失
2. **隐私保护**：选手个人信息的保护，只共享必要信息
3. **退赛流程**：提前说明退赛政策和流程
4. **替补机制**：如果有选手退赛，是否有替补机制
5. **知识产权**：项目知识产权归属需要提前说明
6. **免责条款**：参赛选手需要签署免责声明
7. **差旅补助**：提前说明差旅补助政策和发放方式
8. **组队截止提醒**：组队截止前多次提醒，避免选手错过

## 八、历史教训

> **历史活动 选手管理**：使用飞书多维表格作为一站式系统，涵盖选手报名、组队、项目提交。选手编辑招募信息通过"招募信息发布墙"文档，避免消息被淹没。

> **历史活动 的经验**：项目提交要求通过"最终清单"文档明确，包含所有需要提交的材料和截止时间。赛后调查问卷用于收集选手反馈。
