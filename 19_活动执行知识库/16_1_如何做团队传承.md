### 知识库建设步骤

```
第一步：盘点现有资料
  - 收集团队已有的所有文档、模板、指南
  - 按上述结构分类归档
  - 标记缺失的文档

第二步：补充缺失文档
  - 分配写作任务给各组负责人
  - 设置截止时间（建议2周内完成）
  - 使用统一的文档模板

第三步：审核和优化
  - 由核心团队审核每份文档
  - 确保内容准确、实用
  - 补充"新手视角"的说明

第四步：发布和维护
  - 上传到共享云盘（推荐飞书/Notion/腾讯文档）
  - 设置编辑权限（核心团队可编辑，新人可查看）
  - 指定知识库管理员，定期更新
```

## 第二步：建立AB角机制

### 什么是AB角机制

```
每个关键岗位设置A角和B角：
- A角：主要负责人，承担该岗位的核心工作
- B角：备份负责人，了解A角的工作内容，可随时接替

目的：
- 避免"单点故障"——如果A角因故缺席，B角可无缝接替
- 加速新人成长——B角在协助A角的过程中学习
- 实现平稳交接——换届交接时，B角自然成为新A角
```

### AB角分配方案

| 岗位 | A角（现任） | B角（后备） | 交接时间 | 交接状态 |
|------|-----------|-----------|---------|---------|
| 总负责人 | | | | |
| 赛题组组长 | | | | |
| 宣发组组长 | | | | |
| 赞助组组长 | | | | |
| 场地组组长 | | | | |
| 选手组组长 | | | | |
| 技术组组长 | | | | |
| 志愿者组组长 | | | | |

### AB角工作规范

```
A角职责：
1. 承担该岗位的核心决策和执行工作
2. 定期与B角同步工作进展（建议每周1次）
3. 将重要经验和教训记录到知识库
4. 在重要工作场景中让B角旁观/参与

B角职责：
1. 积极学习A角的工作内容和方法
2. 协助A角完成日常工作
3. 在A角缺席时代行职责
4. 记录学习笔记，补充到知识库

交接规则：
1. 交接期至少1个月（覆盖一个完整的工作周期）
2. 交接内容包括：工作文档、联系人清单、工具账号、经验教训
3. 交接完成后，B角成为新A角，并指定自己的B角
```

## 第三步：新人培训流程

### 新人培训时间线

| 阶段 | 时间 | 内容 | 形式 |
|------|------|------|------|
| 入门期 | 第1周 | 团队介绍、文化了解、知识库浏览 | 线上自学+答疑 |
| 学习期 | 第2-3周 | 岗位知识学习、工具使用培训 | 线上/线下培训 |
| 实践期 | 第4-6周 | 跟随A角参与实际工作 | 实战带教 |
| 独立期 | 第7-8周 | 独立承担小任务，A角辅导 | 独立执行+复盘 |

### 新人培训课程表

| 序号 | 课程 | 时长 | 讲师 | 形式 |
|------|------|------|------|------|
| 1 | 历史活动是什么：历史、使命、价值观 | 30分钟 | 总负责人 | 讲座 |
| 2 | 团队结构和各组职责 | 30分钟 | 总负责人 | 讲座 |
| 3 | 知识库使用指南 | 30分钟 | 知识库管理员 | 实操 |
| 4 | 常用工具培训（协作工具、设计工具等） | 1小时 | 各组代表 | 实操 |
| 5 | 岗位专项培训 | 1-2小时 | A角 | 一对一 |
| 6 | 往届活动案例分析 | 1小时 | 核心团队 | 讨论 |
| 7 | 参与一次模拟任务 | 2小时 | A角 | 实战 |

## 常见问题

**Q1：上一届团队成员都毕业了/离开了怎么办？**
A：①在活动结束时就启动传承工作，不要等到换届；②录制视频版的工作交接；③建立详细的文档弥补人员缺失；④联系上一届成员进行远程答疑。

**Q2：知识库没人维护怎么办？**
A：①指定1名知识库管理员（建议由总负责人兼任）；②每次活动后强制要求各组更新文档；③在团队考核中加入"知识库贡献"指标。

**Q3：新人不愿意看文档怎么办？**
A：①文档要简洁实用，不要写成论文；②配合视频教程；③安排老成员一对一辅导；④设计"入职任务"，引导新人主动查阅文档。

**Q4：AB角机制执行不下去怎么办？**
A：①从最重要的3-5个岗位开始试点；②在团队例会中加入"AB角同步"环节；③对B角的成长给予认可和激励。

## 历史教训

- 历史活动传承的核心是"人+文档"双保险，缺一不可
- 知识库建议使用飞书文档或Notion，方便多人协作和版本管理
- AB角机制是历史活动团队稳定发展的关键，确保每届至少有30%的留存率
- 新人培训建议安排在活动前3-4个月启动，留出足够的学习时间
- 历史教训表明：最好的传承是"一起做一次"——让新人在老成员指导下完整参与一次活动筹备
- 交接文档不要追求完美，重点是"够用"——核心信息传达清楚即可
