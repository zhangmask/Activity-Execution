# 15 总结与复盘

> 本手册适用于活动结束后的总结、复盘和知识沉淀。
> 参考案例：活动主办方 历史活动

---

## 一、核心原则

1. **复盘要趁热打铁**：活动结束后3天内启动复盘，7天内完成
2. **全员参与**：每个组都要复盘，不只是负责人
3. **数据说话**：用数据而非感觉来评估效果
4. **记录教训**：好的和不好的都要记录，为下次活动积累经验

---

## 二、复盘流程

### 2.1 复盘时间线

| 时间节点 | 任务 |
|---|---|
| T+1天 | 物料回收、费用结算、素材收集 |
| T+2天 | 各组内部复盘、填写复盘表 |
| T+3天 | 全员复盘会议 |
| T+5天 | 复盘文档整理 |
| T+7天 | 复盘文档定稿、归档 |

### 2.2 复盘会议议程

```
1. 活动数据总览（10分钟）
   - 参与人数、项目数量、媒体报道等
2. 各组复盘汇报（每组10分钟）
   - 做得好的地方
   - 需要改进的地方
   - 具体案例
3. 自由讨论（20分钟）
   - 跨组协作的问题
   - 突发情况的处理
4. 总结和行动计划（10分钟）
   - 关键改进点
   - 下次活动的建议
```

## 三、复盘内容框架

### 3.1 数据总结

| 维度 | 数据项 | 数据来源 |
|---|---|---|
| 参与规模 | 选手人数、游客人数、嘉宾人数 | 报名系统、签到表 |
| 项目情况 | 项目数量、赛道分布、完成率 | 项目提交系统 |
| 传播效果 | 媒体报道数、社交媒体互动量 | 后台数据 |
| 满意度 | 选手满意度、嘉宾满意度 | 赛后问卷 |
| 财务情况 | 总收入、总支出、结余 | 财务台账 |

## 四、赛后调查问卷

### 4.1 问卷设计

| 维度 | 问题示例 |
|---|---|
| 整体满意度 | 您对本次活动的整体满意度？（1-5分） |
| 赛道体验 | 赛道设置是否合理？评分标准是否清晰？ |
| 场地设施 | 场地设施是否满足需求？网络/电力是否稳定？ |
| 餐饮住宿 | 餐饮质量如何？住宿安排是否满意？ |
| 组织服务 | 工作人员服务态度如何？信息沟通是否及时？ |
| 最佳环节 | 您最喜欢的环节是什么？ |
| 改进建议 | 您认为最需要改进的地方是什么？ |
| 参与意愿 | 您是否愿意参加下次活动？ |

### 4.2 问卷发放

- 活动结束当天通过群聊发放
- 邮件通知所有参与者
- 设置截止时间（建议3天内）
- 提供填写激励（如抽奖）

## 五、知识沉淀

### 5.1 归档内容

| 归档项 | 说明 | 存放位置 |
|---|---|---|
| 活动方案 | 最终版活动方案 | 文档库 |
| 复盘文档 | 各组复盘和总结 | 文档库 |
| 设计素材 | 所有设计源文件 | 素材库 |
| 照片视频 | 活动照片和视频 | 素材库 |
| 财务报告 | 决算表和台账 | 财务库 |
| 联系人信息 | 嘉宾、赞助商、供应商联系方式 | 联系人库 |
| 赛后问卷 | 问卷结果和分析 | 文档库 |

### 5.2 经验教训库

建立结构化的经验教训库：

```
# 经验教训库

## 场地相关
- [经验] 场地确认应尽早，不定则后续工作无法推进
- [教训] 五一期间场地可能不持续供电供冷，需特别确认

## 物料相关
- [经验] 物料清单应尽早定稿，设计稿统一管理
- [教训] 手持KT版等互动道具容易被忽略

## ...
```

- 定期分享活动后续进展
- 邀请参与下次活动

## 七、易遗漏事项

1. **素材收集**：活动结束后及时收集所有摄影摄像素材，避免丢失
2. **设备归还**：借用的设备及时归还
3. **场地恢复**：场地恢复原状，检查有无遗留物品
4. **财务结算**：所有费用及时结算，避免拖延
5. **知识产权**：活动产出的内容（照片、视频）的版权归属确认
6. **数据备份**：所有电子数据备份保存
7. **感谢信**：不要忘记给所有帮助过的人发送感谢信
8. **媒体报道**：整理媒体报道链接，作为活动成果

## 九、历史教训

> **历史活动 复盘**：制作了专门的总结文档，包含内部工作文档汇总、结项报告、各视角反思复盘（负责人C视角、负责人D视角）、未来发展思考等。使用飞书Wiki进行协作文档复盘。

> **历史活动 的教训**：复盘要趁热打铁，不要等活动结束后很久才开始。每个参与者都应该写自己的复盘，而不只是负责人总结。经验教训要结构化记录，方便下次活动参考。
