# 13 游客与观众管理

> 本手册适用于活动游客/观众的报名、接待和现场管理。
> 参考案例：活动主办方 历史活动

---

## 一、核心原则

1. **游客是活动氛围的重要组成**：好的游客体验能提升活动口碑
2. **门禁管理要清晰**：明确哪些区域对游客开放，哪些需要权限
3. **报名要简便**：游客报名流程要比选手简单得多
4. **安全第一**：控制人流，避免安全隐患

---

## 二、游客报名管理

### 2.1 报名表设计

| 信息项 | 必填 | 说明 |
|---|---|---|
| 姓名 | 是 | 用于签到核对 |
| 手机号 | 是 | 发送确认短信 |
| 身份 | 是 | 学生/职场人/其他 |
| 参观日期 | 是 | 选择到场日期 |
| 来源渠道 | 否 | 了解获客渠道 |

### 2.2 报名审核

- 开放报名时间：T-14天
- 报名截止时间：T-3天（或额满即止）
- 现场报名：支持现场扫码登记，但不保证入场

### 2.3 确认通知

报名成功后发送确认短信/消息：
```
[活动名称] 您已成功预约xx月xx日的参观名额。
时间：xx:xx - xx:xx
地点：xxx
请凭此短信到签到处领取参观手环。
如需取消请回复"取消"。
```

---

## 三、门禁管理

### 3.1 区域权限

| 区域 | 游客权限 | 说明 |
|---|---|---|
| 展区 | 可进入 | 凭参观手环 |
| 主会场（开幕式） | 需报名 | 凭短信+手环，严控人数 |
| Workshop区 | 可进入 | 不需要门禁 |
| 开发区 | 禁止进入 | 选手专属区域 |
| 嘉宾休息室 | 禁止进入 | 嘉宾专属区域 |

### 3.2 手环管理

| 手环颜色 | 对应身份 | 权限范围 |
|---|---|---|
| 蓝色 | 选手 | 全区域 |
| 绿色 | 嘉宾/评委 | 全区域+VIP区域 |
| 黄色 | 工作人员 | 全区域+后勤区域 |
| 红色 | 游客 | 展区+开放区域 |
| 白色 | 志愿者 | 工作区域+展区 |

### 3.3 人流控制

- 主会场容量限制：根据场地消防要求设定上限
- 爆满时的分流方案：引导至展区或等候区
- 门口设置：安排志愿者控制入场

---

## 四、游客现场接待

### 4.1 签到流程

1. 游客到达签到处
2. 出示确认短信/现场扫码登记
3. 领取参观手环和活动指南
4. 引导至展区

### 4.2 游客引导

- 设置明显的指引标识
- 安排志愿者在关键节点引导
- 提供活动地图/导览手册
- 设置咨询台，解答游客疑问

### 4.3 游客互动

- 游园会/展会的互动环节
- 拍照打卡点
- 周边礼品/纪念品
- 社交媒体互动（发帖@官方）

---

## 五、易遗漏事项

1. **游客容量限制**：提前确认场地的最大容纳人数，避免超员
2. **安保安排**：游客区域需要安保人员维持秩序
3. **卫生间指引**：游客不熟悉场地，需要明确的卫生间指引
4. **停车场指引**：自驾游客的停车指引
5. **无障碍设施**：残障游客的接待方案
6. **儿童管理**：如果有儿童游客，需要特别注意安全
7. **失物招领**：设置失物招领处
8. **医疗急救**：游客区域的医疗急救点

## 七、历史教训

> **历史活动 游客管理**：使用飞书报名表收集游客信息。游客凭短信领取手环进入开幕式主会场，严控报名才能进入。未报名游客可现场扫码登记，如主会场已满则引导至展区或等候区。

> **历史活动 的经验**：游客细则需要提前发布，明确哪些区域是游客无法进入的。门禁二维码需要提前申请（场地A给二维码，一个二维码只能给50个人用）。
