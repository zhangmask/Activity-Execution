# 如何招募和筛选志愿者

> **适用场景**：hackathon活动需要招募志愿者协助现场执行，包括签到、引导、物料管理、摄影等岗位。
> **前置条件**：已制定志愿者管理计划（参考 09_志愿者管理.md）。

---

## 第一步：分析志愿者需求

### 志愿者需求分析表

| 岗位 | 人数 | 工作内容 | 技能要求 | 工作时段 | 备注 |
|------|------|----------|----------|----------|------|
| 签到组 | 4人 | 来宾签到、发放物料 | 细心、沟通好 | 全程 | 需要早到 |
| 引导组 | 6人 | 路线引导、答疑 | 熟悉场地、耐心 | 全程 | 需要走动 |
| 物料组 | 3人 | 物料搬运、发放、回收 | 力气大、细心 | 前后半天 | 体力活 |
| 摄影组 | 2人 | 活动拍照、录像 | 会用相机/手机拍摄 | 全程 | 有设备优先 |
| 技术支持组 | 2人 | 网络、设备调试 | 有IT基础 | 全程 | 优先CS专业 |
| 嘉宾对接组 | 2人 | 嘉宾接待、引导 | 形象好、沟通强 | 嘉宾到场时段 | 优先女生 |
| 餐饮组 | 2人 | 餐食分发、餐饮区管理 | 细心、卫生意识 | 用餐时段 | - |
| 机动组 | 3人 | 随时支援各岗位 | 灵活、反应快 | 全程 | 需要体力好 |

**志愿者总需求**：约24人（根据活动规模调整）

---

## 第二步：确定招募渠道

### 招募渠道对比

| 渠道 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| **校内社团/学生会** | 人员素质高、组织性强 | 需要社团负责人协助 | 首选渠道 |
| **志愿者平台** | 人员充足 | 素质参差不齐 | 大型活动补充 |
| **往届志愿者** | 有经验、上手快 | 数量有限 | 优先邀请 |
| **朋友圈/社交群** | 传播快 | 可控性差 | 辅助渠道 |
| **活动现场报名者** | 热情高 | 经验不足 | 最后补充 |
| **合作院校** | 跨校合作 | 沟通成本高 | 跨校活动 |

### 招募时间节点

- **活动前4周**：发布招募信息
- **活动前3周**：截止报名
- **活动前2周**：完成筛选和通知
- **活动前1周**：完成培训

## 第四步：设计报名表

### 志愿者报名表字段设计

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 文本 | 是 | 真实姓名 |
| 性别 | 单选 | 是 | 用于分组安排 |
| 手机号 | 文本 | 是 | 联系方式 |
| 微信号 | 文本 | 是 | 拉群用 |
| 学校/单位 | 文本 | 是 | - |
| 年级/专业 | 文本 | 否 | 了解背景 |
| 岗位偏好 | 多选 | 是 | 第一志愿+第二志愿 |
| 相关经验 | 长文本 | 否 | 志愿者/活动经验 |
| 可用时间 | 多选 | 是 | Day1上午/Day1下午/Day1晚上/Day2 |
| 特殊技能 | 文本 | 否 | 摄影/IT/设计等 |
| 是否参加过往届 | 单选 | 否 | 优先录取往届志愿者 |
| T恤尺码 | 单选 | 是 | S/M/L/XL/XXL |
| 自我介绍 | 长文本 | 否 | 了解个人特点 |

## 第五步：筛选和录用

### 筛选标准

| 筛选维度 | 权重 | 评估要点 |
|----------|------|----------|
| 可用时间 | 30% | 是否覆盖活动关键时段 |
| 相关经验 | 25% | 是否有志愿者或活动组织经验 |
| 岗位匹配 | 20% | 技能是否与岗位匹配 |
| 沟通能力 | 15% | 报名表填写是否认真、表达是否清晰 |
| 往届经历 | 10% | 是否参加过往届活动 |

### 筛选流程

1. **初步筛选**：排除信息不完整、不可用时间的申请
2. **打分排序**：按筛选标准给每位申请者打分
3. **岗位匹配**：根据岗位需求和申请者偏好进行匹配
4. **备选名单**：录取人数的120%作为候选，多出的作为备选
5. **发送通知**：发送录用通知和未录用感谢通知

## 常见问题

**Q：报名人数不足怎么办？**
A：①扩大招募渠道（增加校园海报、朋友圈转发）；②降低筛选标准；③联系合作院校支援；④从选手报名者中招募（非参赛选手）。

**Q：志愿者临时来不了怎么办？**
A：①提前建立备选名单（多招20%）；②活动前一天再次确认出席；③准备机动岗位可临时顶替。

**Q：志愿者是无偿的吗？**
A：建议提供基本权益：志愿者证书、活动周边、免费餐饮、社会实践时长。这些成本不高但能大大提升志愿者的积极性和满意度。

**Q：如何避免志愿者只来半天就走？**
A：①在招募时明确工作时段要求；②建立签到签退制度；③承诺完成全程才能获得证书；④让志愿者感受到价值和归属感。

## 历史教训

在hackathon活动中，志愿者招募建议：
- **人数参考**：200人规模的hackathon建议20-30名志愿者
- **核心岗位**：签到组和引导组是最重要的，一定要安排靠谱的人
- **技术志愿者**：hackathon特有的"技术支援"岗位，帮选手解决网络、环境配置问题，很受欢迎
- **志愿者来源**：与学校计算机社团、青年志愿者协会合作是最稳定的渠道
- **激励机制**：除了证书，还可以让志愿者参与活动的after party，增强归属感
