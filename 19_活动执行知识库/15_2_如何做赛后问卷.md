# 如何做赛后问卷

> **适用场景**：黑客松活动结束后，收集选手、嘉宾、观众和工作人员的反馈。
> **前置条件**：请先完成 `15_1_如何开复盘会.md`，了解复盘框架和数据需求。

---

## 第一步：确定问卷维度和设计问题

### 问卷维度框架

| 维度 | 目的 | 问题数量 |
|------|------|---------|
| 整体满意度 | 了解总体评价 | 2-3题 |
| 活动内容 | 评估赛题、环节设计 | 3-4题 |
| 组织执行 | 评估流程、服务、现场 | 3-4题 |
| 场地设施 | 评估场地、设备、餐饮 | 2-3题 |
| 导师/评委 | 评估指导和评审质量 | 2-3题 |
| 社交体验 | 评估交流、社交价值 | 2-3题 |
| 改进建议 | 收集具体建议 | 2-3题 |
| 未来意愿 | 评估复参/推荐意愿 | 1-2题 |

### 选手问卷问题示例

## 第二步：设计问卷发放方式

### 不同人群的问卷版本

| 人群 | 问卷版本 | 问题数量 | 发放时间 | 预计完成时间 |
|------|---------|---------|---------|-------------|
| 选手 | 完整版 | 25-30题 | 活动结束后当天 | 10-15分钟 |
| 嘉宾/评委 | 精简版 | 10-15题 | 活动结束后1天 | 5-8分钟 |
| 观众/游客 | 快速版 | 5-8题 | 离场时扫码 | 2-3分钟 |
| 工作人员 | 内部版 | 15-20题 | 活动结束后2天 | 8-10分钟 |

### 发放渠道

| 渠道 | 操作方式 | 优势 |
|------|---------|------|
| 微信群 | 在选手群/志愿者群发问卷链接 | 到达率高 |
| 邮件 | 发送问卷链接到报名邮箱 | 正式、可追踪 |
| 短信 | 发送问卷链接短信 | 覆盖面广 |
| 现场二维码 | 在出口处/资料袋中放置二维码 | 趁热打铁、回收率高 |
| 社交媒体 | 在公众号/微博发问卷链接 | 覆盖广泛 |

### 发放时间策略

```
最佳发放时间：
- 选手：活动结束当天晚上（趁记忆清晰）
- 嘉宾：活动结束后1天（避免太打扰）
- 观众：离场时现场扫码（回收率最高）
- 工作人员：活动结束后2-3天（休息后更理性）

提醒策略：
- 首次发放后3天，对未填写者发一次提醒
- 问卷开放时间建议为7天
- 设置填写激励（抽奖、小礼品）
```

## 第三步：数据分析方法

### 量化数据分析

#### 满意度分析

```python
# 数据分析框架（以Excel为例）
1. 计算各维度平均分
2. 绘制雷达图展示各维度得分
3. 与上届数据对比（如有）
4. 识别最高分和最低分维度
```

#### 满意度分析模板

| 维度 | 平均分 | 最高分 | 最低分 | 标准差 | 上届分数 | 变化 |
|------|--------|--------|--------|--------|---------|------|
| 整体满意度 | | | | | | |
| 活动内容 | | | | | | |
| 组织执行 | | | | | | |
| 场地设施 | | | | | | |
| 导师/评委 | | | | | | |
| 社交体验 | | | | | | |

#### NPS（净推荐值）计算

```
NPS = 推荐者比例(9-10分) - 贬损者比例(0-6分)

NPS > 50：优秀
NPS 30-50：良好
NPS 0-30：需要改进
NPS < 0：严重问题
```

#### 交叉分析

| 分析维度 | 交叉方式 | 目的 |
|---------|---------|------|
| 满意度 × 身份类型 | 选手vs观众满意度差异 | 了解不同人群需求 |
| 满意度 × 参与天数 | 多天vs单天参与者 | 优化活动时长 |
| 满意度 × 经验水平 | 新手vs老手 | 针对性改进 |
| 满意度 × 团队规模 | 个人vs团队参赛 | 优化组队机制 |
