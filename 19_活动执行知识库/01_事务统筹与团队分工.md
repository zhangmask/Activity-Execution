# 01 事务统筹与团队分工

> 本手册适用于任何大型活动（黑客松、Demo Day、创业集市等）的事务统筹和团队分工。
> 参考案例：活动主办方 历史活动

---

## 一、核心原则

1. **一件事只有一个总负责人**：所有事项必须明确到唯一的主负责人，避免"都觉得有人在管"的真空地带
2. **一个人原则上不跨多个组**：减少沟通切换成本，但允许机动组成员临时支援
3. **分工文档集中在一个地方**：不要在多个文档分别分工，统一在一个总表中维护
4. **所有分工对应到主负责人**，而非"某某团队负责"——落到人头才有执行力

---

## 二、团队架构模板

### 建议分组（可按活动规模增减）

| 小组 | 核心职责 | 建议人数 |
|---|---|---|
| **总协调组** | 盯紧各组进度、调配人员、应急决策 | 2-3人 |
| **策划执行板块** | 活动方案设计、流程把控、场地执行 | 4-6人 |
| **宣传设计板块** | 视觉设计、宣发文案、摄影摄像、直播 | 4-6人 |
| **对外合作板块** | 赞助商对接、嘉宾邀请、合作方管理 | 3-5人 |
| **技术板块** | 现场设备、线上系统、网络保障 | 2-3人 |
| **财务板块** | 预算管理、报销、台账 | 1-2人 |
| **机动组** | 巡视、应急补充、临时支援 | 3-5人 |

> **历史教训**：早期分工文档搞太多了，分散在多个地方，后期应全部集中在一个文档维护。

## 四、关键节点 Checklist

### 筹备启动期（T-60 ~ T-45天）

- [ ] 确定活动名称、主题、规模
- [ ] 组建核心团队，明确各组负责人
- [ ] 初步预算框架（场地、物料、嘉宾、餐饮、宣传）
- [ ] 场地初步踩点和意向确认
- [ ] 建立统一的文档协作空间（飞书/Notion等）

### 框架搭建期（T-45 ~ T-30天）

- [ ] 完成对外介绍文档（面向选手、赞助商、嘉宾的版本）
- [ ] 启动主视觉设计
- [ ] 预算表细化到每个科目的预估金额
- [ ] 赞助合作方案和权益表定稿

### 外联资源期（T-30 ~ T-20天）

- [ ] 发布选手招募信息
- [ ] 完成核心嘉宾邀请
- [ ] 展商/合作方确认
- [ ] 选手报名系统上线

### 细化落地期（T-20 ~ T-10天）

- [ ] 所有活动环节设计定稿
- [ ] 志愿者招募完成并分组
- [ ] 物料设计稿完成、下单制作
- [ ] 住宿和交通方案确认

### 冲刺执行期（T-10 ~ T-3天）

- [ ] 所有宣传物料到位
- [ ] 现场物料清单核对
- [ ] 全流程彩排（含设备测试）
- [ ] 应急预案制定（天气、设备故障、人员缺席等）
- [ ] 所有电子文件收齐（PPT、视频、BGM、流程表、座位图）

## 五、易遗漏事项

1. **分工文档分散**：初期容易在多个地方分工，导致信息不一致。务必从一开始就集中维护
2. **人员时间冲突**：提前收集每个人可参与的时间段，标注不在场时间
3. **政府报备**：超过一定人数的集会需要向公安报备，大型活动需提前15天向相关部门提交申请
4. **应急资金**：预留10%-15%的预算应对突发需求
5. **联络信息表**：所有工作人员的电话号码必须提前收集，现场通讯录人手一份
6. **场地审批风险**：提前确认场地使用许可，准备备用场地方案
7. **转场时间**：多场地活动要考虑交通接驳时间，提前规划大巴路线

## 七、历史教训

> **历史活动 的教训**：分工类文档搞太多了，后续 S3 计划全部集中在一个文档。建议从一开始就只维护一份总分工表，其他文档引用这份总表即可。

> **历史活动 的经验**：早期用"团队分工与时间线"作为核心文档，后期切换到"活动方案"文档做统筹。两个文档之间的衔接容易出现信息断层，建议用一个文档贯穿始终。
