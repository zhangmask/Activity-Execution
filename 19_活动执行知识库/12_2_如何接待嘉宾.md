# 如何接待嘉宾

> **适用场景**：hackathon活动当天需要接待嘉宾，确保嘉宾有良好的参与体验。
> **前置条件**：已完成嘉宾邀请和信息收集（参考 12_1_如何邀请嘉宾）。

---

## 第一步：了解接待流程

### 嘉宾接待全流程

```
到达 → 签到 → 休息 → 候场 → 上台 → 合影 → 用餐 → 离开
```

### 每个环节的详细安排

| 环节 | 时间 | 地点 | 负责人 | 具体事项 |
|------|------|------|--------|----------|
| **到达** | 提前30分钟 | 场地入口 | 接站人员 | 接站/迎接，引导至嘉宾休息室 |
| **签到** | 到达后5分钟 | 签到处 | 签到组 | 核对信息，发放嘉宾物料包 |
| **休息** | 签到后 | 嘉宾休息室 | 嘉宾对接人 | 提供饮品、介绍日程、确认演讲内容 |
| **候场** | 上台前10分钟 | 候场区 | 嘉宾对接人 | 引导至候场区，介绍上台流程 |
| **上台** | 按日程 | 主会场 | 主持人 | 主持人介绍→嘉宾上台→演讲/分享 |
| **合影** | 上台后 | 主会场/指定区域 | 摄影组 | 与主办方/选手合影 |
| **用餐** | 用餐时段 | VIP用餐区 | 餐饮组 | 引导至嘉宾用餐区 |
| **离开** | 活动结束/嘉宾行程 | 场地出口 | 嘉宾对接人 | 感谢、送别、安排车辆 |

- [ ] 提前打印嘉宾胸牌（核对姓名和头衔）

- [ ] 检查所有物料是否齐全
- [ ] 按嘉宾姓名标签分装
- [ ] 提前放置在嘉宾休息室或签到处

## 第四步：安排嘉宾对接人

### 嘉宾对接人安排表

| 嘉宾 | 对接人 | 联系方式 | 到达时间 | 接站/迎接 | 备注 |
|------|--------|----------|----------|-----------|------|
| 张三 | 小王 | 138xxxx | 6/15 14:00 | 高铁站接站 | 开幕嘉宾 |
| 李四 | 小李 | 139xxxx | 6/15 08:30 | 场地迎接 | 导师嘉宾 |
| 王五 | 小张 | 137xxxx | 6/15 13:00 | 场地迎接 | 评审嘉宾 |

### 嘉宾对接人职责

**活动前**：
- 与嘉宾保持联系，确认行程
- 提前了解嘉宾需求和习惯
- 准备嘉宾物料包
- 确认演讲内容和PPT

**活动中**：
- 全程陪同嘉宾（或随时可联系）
- 引导嘉宾到各个活动环节
- 协助嘉宾解决突发需求
- 确保嘉宾演讲设备正常
- 安排嘉宾合影

**活动后**：
- 送别嘉宾
- 安排车辆（如需要）
- 发送感谢信息
- 分享活动照片

### 嘉宾对接人行为规范

- **提前到达**：比嘉宾早到30分钟
- **手机畅通**：保持手机开机，随时可联系
- **主动服务**：主动询问需求，不要等嘉宾开口
- **尊重隐私**：不随意拍照、不索要私人联系方式
- **专业态度**：保持礼貌、专业、热情

## 第五步：嘉宾演讲对接

### 演讲前对接checklist

- [ ] 提前1周收到嘉宾PPT
- [ ] 检查PPT格式（确认是否需要转换）
- [ ] 确认演讲时长
- [ ] 确认设备需求（投影、翻页笔、麦克风）
- [ ] 提前测试PPT在会场设备上的显示效果
- [ ] 准备备用方案（如PPT无法播放，准备PDF版本）

### 演讲当天流程

```
嘉宾到达休息室（演讲前30分钟）
    ↓
确认PPT和设备（演讲前15分钟）
    ↓
引导至候场区（演讲前10分钟）
    ↓
主持人介绍嘉宾（演讲前1分钟）
    ↓
嘉宾上台演讲
    ↓
演讲结束 → 引导回休息室/合影区
```

### 主持人介绍模板

```
下面有请[嘉宾姓名]老师为我们带来分享！

[嘉宾姓名]老师是[公司/机构]的[职位]，[简要介绍嘉宾背景和成就]。

今天他/她将为我们分享[演讲主题]。

掌声有请！
```

## 第六步：嘉宾送别

### 送别流程

1. **活动结束前确认**：提前与嘉宾确认离开时间
2. **安排车辆**：如需要，提前预约网约车或安排专车
3. **赠送礼品**：如准备了额外礼品，在送别时赠送
4. **合影留念**：在嘉宾离开前安排合影
5. **表达感谢**：当面再次感谢嘉宾的参与
6. **送至门口**：送嘉宾到车辆/出口

### 活动后感谢模板（微信/邮件）

```
尊敬的[嘉宾姓名]老师：

您好！

感谢您百忙之中参加[活动名称]！您的[演讲/指导/评审]让选手们受益匪浅，活动反馈非常好。

📎 附上：
1. 活动照片（含您的精彩瞬间）
2. 活动回顾推文链接
3. 获奖项目名单

再次感谢您的支持！期待未来有更多合作机会。

祝工作顺利！

[你的姓名]
[活动组委会]
```

## 常见问题

**Q：嘉宾迟到怎么办？**
A：①提前联系确认行程；②如果迟到，调整日程（将其他环节提前）；③准备替补嘉宾或主持人串场方案。

**Q：嘉宾演讲超时怎么办？**
A：①提前与嘉宾沟通时间限制；②安排主持人在剩余5分钟时举牌提醒；③主持人礼貌打断并引导进入Q&A环节。

**Q：嘉宾对场地/设备不满意怎么办？**
A：①提前检查设备，尽量提前解决问题；②准备备用设备（备用投影、备用翻页笔）；③安排技术人员随时待命。

**Q：多位嘉宾同时到达怎么接待？**
A：①提前安排足够数量的对接人；②如果对接人不够，优先接待最重要的嘉宾；③其他嘉宾引导至休息室，由休息室工作人员接待。

### 真实案例：历史活动嘉宾接送Check流程

| 时间点 | 动作 | 负责人 |
|---|---|---|
| T-1天 | 向嘉宾发送信息包：地点、楼层、联系人、入场方式、停车/打车建议 | 嘉宾接待负责人 |
| T-3小时 | 再次确认嘉宾出发时间、交通方式、预计到达时间 | 嘉宾接待负责人 |
| T-60分钟 | 嘉宾接待到达入口或指定接驳点 | 嘉宾接待+支持人员 |
| T-45分钟 | 嘉宾到场后带至候场区，完成饮水、休息、流程说明 | 嘉宾接待+支持人员 |
| T-30分钟 | 完成设备检查：PPT、投屏、视频、麦克风、翻页笔 | 设备负责 |
| T-10分钟 | 主持人与嘉宾确认上场口径、称谓、讲述时长、Q&A安排 | 主持 |
| T-3分钟 | 嘉宾在上场区待命 | 会场执行 |
| 结束后 | 引导嘉宾合影、采访或离场 | 主持 |

**发给嘉宾信息包（8项）**：
1. 活动名称
2. 场地地址和楼层
3. 建议到达时间（正式开始前45分钟）
4. 现场联系人姓名、电话、微信
5. 交通建议（打车定位、停车信息、入口位置）
6. 技术要求（PPT比例16:9、视频格式、是否需自带电脑）
7. 流程安排（候场→Tech Check→上台→Q&A→合影→离场）
8. 嘉宾权益（餐饮、休息区、礼品、住宿或交通报销说明）

**嘉宾行程确认字段**：姓名、手机号/微信、所属机构/职务、Workshop主题、到达城市时间、到达场地时间、接送方式、接送联系人、PPT是否收到、PPT版本、是否有音频/视频、是否需要转接头（HDMI/Type-C）、是否需要手持麦/领夹麦、饮食/过敏/特殊需求、离场时间与去向

## 历史教训

在hackathon活动中，嘉宾接待建议：
- **导师接待**：hackathon的导师通常需要在开发阶段巡回指导，安排好导师的指导时间段和区域
- **评审嘉宾**：评审环节节奏紧凑，确保评审嘉宾提前了解评分标准和流程
- **轻松氛围**：hackathon氛围通常比较轻松，不需要太正式的接待流程
- **嘉宾也是参与者**：鼓励嘉宾融入活动，与选手互动，而不仅仅是"做嘉宾"
- **赛后关系维护**：活动后保持与嘉宾的联系，为下次活动合作打好基础
