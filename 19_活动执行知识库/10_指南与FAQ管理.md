# 10 指南与FAQ管理

> 本手册适用于活动各类指南、FAQ文档的制作和维护。
> 参考案例：活动主办方 历史活动

---

## 一、核心原则

1. **指南要提前发布**：至少在活动前7天发布，给参与者充足准备时间
2. **FAQ要持续更新**：根据群聊和私信中的新问题持续补充
3. **一个文档解决一类人的问题**：不要把所有信息塞进一个文档
4. **信息要准确一致**：不同指南中的信息不能矛盾

---

## 二、指南体系设计

### 2.1 按受众分类

| 指南类型 | 目标受众 | 核心内容 | 发布时间 |
|---|---|---|---|
| 选手终极指南 | 参赛选手 | 报名、组队、赛制、奖项、交通、住宿 | T-14天 |
| 参展方指南 | 展商/合作方 | 展位、物流、布展、权益 | T-14天 |
| 嘉宾指引手册 | 嘉宾/评委 | 日程、交通、住宿、着装、演讲要求 | T-7天 |
| 导师指引手册 | 导师 | 辅导时间、方式、选手信息 | T-7天 |
| 观察员指南 | AI产品观察员 | 角色说明、观察维度、权益 | T-7天 |
| 游客指南 | 观众 | 时间、地点、报名、交通 | T-7天 |
| 志愿者手册 | 志愿者 | 岗位职责、时间、注意事项 | T-5天 |
| 工作人员手册 | 内部团队 | 分工、流程、应急预案 | T-3天 |

### 2.2 指南通用结构

```
# [角色]指南 - [活动名称]

## 一、活动概况
- 时间、地点、规模
- 活动日程概览

## 二、你需要知道的
- [角色]专属信息
- 重要时间节点
- 需要提前准备的事项

## 三、交通指引
- 如何到达场地
- 停车信息
- 接站安排（如有）

## 四、住宿安排（如有）
- 住宿地点
- 入住/退房时间
- 注意事项

## 五、常见问题 FAQ
- Q1: ...
- Q2: ...

## 六、紧急联系
- 联系人姓名、电话
- 紧急情况处理流程
```

## 三、FAQ管理流程

### 3.1 FAQ收集渠道

| 渠道 | 收集方式 | 频率 |
|---|---|---|
| 选手群 | 监控群聊中的提问 | 每天 |
| 私信 | 一对一咨询记录 | 实时 |
| 报名表 | 报名时的疑问 | 每天 |
| 邮件 | 正式咨询 | 实时 |

### 3.2 FAQ分类整理

| 类别 | 示例问题 |
|---|---|
| 报名相关 | 如何报名？可以跨校组队吗？ |
| 赛制相关 | 可以同时报多条赛道吗？评分标准是什么？ |
| 交通相关 | 怎么到达场地？有停车场吗？ |
| 住宿相关 | 提供住宿吗？需要自带什么？ |
| 设备相关 | 提供电脑吗？网络怎么样？ |
| 餐饮相关 | 包餐吗？有素食吗？ |
| 奖金相关 | 奖金怎么发放？需要交税吗？ |

### 3.3 FAQ维护规范

- 每个问题的回答要简洁明确
- 包含具体的时间、地点、金额等信息
- 如果信息有变更，及时更新FAQ并通知
- FAQ文档顶部标注"最后更新时间"

## 四、群聊管理

### 4.1 群公告模板

```
【公告】[活动名称] 重要通知
时间：xx月xx日
---
各位[角色]，请注意以下事项：
1. [事项1]
2. [事项2]
3. [事项3]
---
详情请查看：[文档链接]
如有疑问请在群内提问或私信管理员。
```

### 4.2 群规则

- 置顶群公告和重要文档链接
- 新人入群先看群公告和FAQ
- 禁止发布与活动无关的广告
- 鼓励提问，但先看FAQ是否已有答案

### 4.3 常用通知模板

| 场景 | 通知内容 |
|---|---|
| 报名确认 | 您的报名已收到，请查收确认邮件 |
| 组队提醒 | 组队截止时间为xx月xx日，请尽快完成 |
| 材料提交 | 项目提交截止时间为xx月xx日xx:xx |
| 日程变更 | xx环节时间调整为xx:xx，地点变更为xxx |
| 紧急通知 | [紧急] xxx，请立即查看 |

## 五、易遗漏事项

1. **信息一致性**：不同文档中的时间、地点、联系人信息要保持一致
2. **版本控制**：FAQ更新后标注版本号和更新时间
3. **多语言**：如果有海外参与者，需要准备英文版本
4. **无障碍**：指南文档要考虑视觉障碍者的阅读需求
5. **离线版本**：准备PDF版本，方便离线查看
6. **二维码**：将重要文档生成二维码，方便现场扫码查看
7. **反馈渠道**：指南中要提供反馈渠道，收集改进建议

## 七、历史教训

> **历史活动 指南体系**：分别制作了选手终极指南、参展方指南、群聊FAQ、赛事活动FAQ、导师嘉宾指引手册、AI产品观察员指南等多份文档。使用飞书文档统一管理，通过链接分享。

> **历史活动 的经验**：群聊&常见问题FAQ需要持续更新，赛事活动FAQ需要覆盖活动期间的各个环节。指南文档要提前发布，给参与者充足的准备时间。
